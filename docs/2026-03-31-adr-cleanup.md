## 感情的な変化

- ADR のフォーマット統一は地味な作業だが、後から読み返したときに一貫性があると気持ちよい。
- 「決定を先に書く」という方針は、世の中の有名フォーマット（Nygard、LADR）を調べた上で選んだので納得感がある。

## 技術的な学び

- 主要な ADR フォーマットの比較
  - **Nygard / LADR**: Context → Decision → Consequences のシンプルな構成。決定が先。
  - **MADR**: Considered Options → Decision Outcome の順。検討プロセスの記録を重視。
  - **Y-Statements**: 文章形式で意思決定を1文にまとめるスタイル。
- 今回は「参照時の読みやすさ」を優先して Nygard/LADR スタイルを採用した。
