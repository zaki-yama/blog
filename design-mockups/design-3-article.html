2 Server Components

Server Componentsは、サーバーサイドでレンダリングされるReactコンポーネントです。 従来のクライアントサイドレンダリングとは根本的に異なるアプローチで、 パフォーマンスとSEOの両面で大幅な改善をもたらします。

app/blog/[slug]/page.tsx 
// Server Component - サーバーサイドで実行
export default async function BlogPost({ params }: { params: { slug: string } }) {
  // データベースやAPIから直接データを取得
  const post = await fetchPostFromDatabase(params.slug);
  const comments = await fetchCommentsFromDatabase(post.id);
  
  return (
    <article className="max-w-4xl mx-auto p-6">
      <header className="mb-8">
        <h1 className="text-4xl font-bold mb-4">{post.title}</h1>
        <div className="flex items-center gap-4 text-gray-600">
          <time>{formatDate(post.publishedAt)}</time>
          <span>{post.readingTime}分で読める</span>
        </div>
      </header>
      
      <div 
        className="prose prose-lg max-w-none mb-12"
        dangerouslySetInnerHTML={{ __html: post.content }}
      />
      
      {/* Client Component - インタラクティブな機能 */}
      <CommentSection postId={post.id} initialComments={comments} />
    </article>
  );
}

主な利点

  • 初期ページロードの高速化
  • JavaScriptバンドルサイズの削減
  • SEO最適化の自動化
  • サーバーリソースへの直接アクセス

考慮事項

  • ブラウザAPIは使用不可
  • イベントハンドラーは設定不可
  • Hookの一部は制限あり
  • 適切な境界設計が重要

3 Suspenseの活用

Suspenseを使用することで、非同期処理中のローディング状態を宣言的に管理できます。 従来のuseEffectとローディング状態の組み合わせから、より直感的で保守しやすいコードに変わります。

components/Dashboard.tsx 
function UserDashboard() {
  return (
    <div className="dashboard-container">
      <header className="mb-8">
        <h1 className="text-3xl font-bold">ダッシュボード</h1>
      </header>
      
      {/* ユーザー情報セクション */}
      <Suspense 
        fallback={<UserProfileSkeleton />}
        key="user-profile"
      >
        <UserProfile />
      </Suspense>
      
      {/* メインコンテンツエリア */}
      <div className="grid lg:grid-cols-2 gap-8 mt-8">
        <Suspense 
          fallback={<PostsGridSkeleton />}
          key="user-posts"
        >
          <UserPosts />
        </Suspense>
        
        <Suspense 
          fallback={<AnalyticsSkeleton />}
          key="analytics"
        >
          <AnalyticsWidget />
        </Suspense>
      </div>
      
      {/* ネストしたSuspense境界の例 */}
      <Suspense fallback={<div>コメントを読み込み中...</div>}>
        <RecentComments />
        <Suspense fallback={<div>返信を読み込み中...</div>}>
          <CommentReplies />
        </Suspense>
      </Suspense>
    </div>
  );
}

Suspenseのベストプラクティス

適切な境界設定
  • • 独立したデータソースごとに境界を分ける
  • • ネストした境界で段階的ローディング
  • • エラーバウンダリとの組み合わせ
UX向上のテクニック
  • • スケルトンローダーの活用
  • • 段階的なコンテンツ表示
  • • 適切なローディング時間の設定

4 パフォーマンス最適化

React 18では並行レンダリング（Concurrent Rendering）の導入により、 ユーザーインタラクションの応答性が大幅に改善されました。 重要な更新と非重要な更新を区別し、優先度に基づいて処理を行います。

useTransition

重い更新処理を非緊急として扱い、ユーザーインタラクションの応答性を保持します。

const [isPending, startTransition] = useTransition();
startTransition(() => setQuery(value));

useDeferredValue

値の更新を遅延させ、高頻度の更新によるパフォーマンス低下を防止します。

const deferredQuery = useDeferredValue(searchQuery);
return <SearchResults query={deferredQuery} />;

6 まとめ

React 18の新機能は、フロントエンド開発における新しい可能性を開きました。 Server ComponentsとSuspenseの組み合わせにより、パフォーマンスとユーザー体験の 両方を大幅に向上させることができます。

これらの機能を実際のプロジェクトで活用することで、より効率的で 保守性の高いアプリケーションを構築できるようになります。 ぜひ段階的に導入を検討してみてください。
